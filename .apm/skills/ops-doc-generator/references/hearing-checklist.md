# ヒアリングチェックリスト

ドキュメント生成に必要な情報の一覧です。
リポジトリから自動収集できる項目は `[自動]` マーク付き。それ以外はユーザーに確認します。

---

## 1. プロジェクト基本情報

- [ ] クライアント名（組織名）
- [ ] プロジェクト名 / サイト名
- [ ] サイトの種類（コーポレート / 採用 / EC / メディア / その他）
- [ ] 運用会社名（開発・保守担当）
- [ ] ドキュメントの主な読者（クライアント担当者 / 社内運用チーム / 両方）

## 2. サイト一覧

- [ ] 本番 URL（サイトごと）
- [ ] ステージング URL（サイトごと）
- [ ] ステージングのアクセス制限方式（Basic 認証 / IP 制限 / Cloudflare Access / Vercel Password Protection 等）
- [ ] アクセス制限のメンバー追加手順

## 3. 利用サービス

以下のカテゴリごとに、使用しているサービスを確認します。

### ドメイン
- [ ] ドメインレジストラ（XDomain / お名前.com / Google Domains / Cloudflare Registrar 等）
- [ ] 管理者（自社 / 別会社）
- [ ] 管理対象ドメイン一覧
- [ ] ネームサーバーの向き先

### ホスティング / CDN
- [ ] `[自動]` ホスティングサービス（Cloudflare Workers / Vercel / AWS / さくら等）
- [ ] プラン・料金
- [ ] 契約アカウント（メールアドレス）
- [ ] 管理画面 URL

### DNS
- [ ] DNS 管理サービス（ホスティングと同じ場合が多い）

### CMS
- [ ] `[自動]` CMS の種類（microCMS / WordPress / Contentful / Strapi 等）
- [ ] プラン・料金
- [ ] 契約アカウント
- [ ] 管理画面 URL（サイトごと）
- [ ] 主なコンテンツ種別（ニュース / ブログ / 求人 等）

### メール送信
- [ ] `[自動]` メール送信サービス（Resend / SendGrid / Amazon SES 等）
- [ ] プラン・料金
- [ ] 契約アカウント
- [ ] 送信元アドレス（サイト・環境ごと）
- [ ] 通知先アドレス（ステージング / 本番、フォーム種別ごと）

### ソースコード管理
- [ ] `[自動]` リポジトリホスティング（GitHub / GitLab / Bitbucket）
- [ ] Organization / グループ名
- [ ] リポジトリ URL（サイトごと）

### セキュリティ / フォーム対策
- [ ] CAPTCHA サービス（Cloudflare Turnstile / reCAPTCHA / hCaptcha）
- [ ] レート制限の有無・設定

### 外部連携
- [ ] 求人情報連携（OwnedMaker / Indeed 等）
- [ ] SNS 連携（Instagram / X 等）
- [ ] アクセス解析（Google Analytics / Plausible 等）
- [ ] その他の外部 API

### 通知・監視
- [ ] アラート設定の有無（Cloudflare / Vercel / UptimeRobot 等）
- [ ] 通知先メールアドレス
- [ ] 監視対象（使用量 / 障害 / メンテナンス）

## 4. ランニングコスト

- [ ] 各サービスのプラン・月額費用
- [ ] 従量課金の有無・目安

## 5. 権限・アカウント管理

- [ ] サービスごとのアカウント運用方針（個別招待 / 共通アカウント）
- [ ] メンバー追加・削除の手順（各サービス）

## 6. コンテンツ更新

- [ ] CMS での記事投稿フロー
- [ ] プレビュー機能の有無・使い方
- [ ] 公開後の自動デプロイの仕組み
- [ ] デプロイ状況の確認方法

## 7. デプロイ

- [ ] `[自動]` デプロイ方式（GitHub Actions / Cloudflare Pages / Vercel 等）
- [ ] ブランチ戦略（main = 本番, develop = ステージング 等）
- [ ] コード変更時のデプロイフロー

## 8. 依存関係の更新

- [ ] `[自動]` Dependabot / Renovate の設定有無
- [ ] 更新 PR の対応フロー・頻度

## 9. 障害対応

- [ ] 各サービスのステータスページ URL
- [ ] エスカレーションフロー（誰が一次対応するか）
- [ ] 対応体制（担当会社・連絡先）
