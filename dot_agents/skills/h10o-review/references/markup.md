# markup review rules

## 必須チェック（優先）

- 見出し階層（`h1` 原則1つ、レベル飛びなし）
- 画像 `alt`（意味あり: 説明、装飾: `alt=""`）
- `img` の `width` / `height`（CLS抑制）
- キーボード操作可能性（Tab到達、主要操作実行）
- `title` / `meta description` の固有化

### 検出シグナル

- 見出しがデザイン都合でスキップ（`h2` の次が `h4` など）
- クリック可能要素が `div` / `span` のみで実装
- フォーカス可視化が消えている（`outline: none` のみ）
- 同一ページで `title` / description がテンプレ固定
- 画像寸法未指定で初期描画時にレイアウトが動く
- `<br>` が段落間スペース・余白確保・リスト的並びに使われている（`<p>`, `<ul>`, CSS余白で代替すべき）
- ネイティブ要素が暗黙に持つロールと同じ `aria-*` が付与されている（二重読み上げの原因）

## 推奨チェック

- セマンティック要素の利用（`main`, `nav`, `section` など）
- `<br>` が視覚的レイアウト目的で使われていないか（余白・段落分け・リスト表現に `<br>` を使わない）
- ネイティブHTMLで意味が伝わる箇所に冗長な `aria-*` を付けていないか（例: `<button aria-role="button">`, `<nav aria-label="navigation">` など）
- リンク/ボタンの使い分け
- `target="_blank"`: 主要ブラウザ（2021年〜）は暗黙的に `noopener` を適用するため `rel="noopener"` の明示は不要。`noreferrer` はサイト外リンクでは推奨だが、サイト内リンクに付けるとリファラー情報が欠損しアクセス解析に影響するため使い分けが必要
- メディア最適化（`picture`, `srcset`, `loading`）
- コントラスト（WCAG AA 目安）

### A11Y詳細観点

- フォーム要素にラベルが関連付けられているか
- エラーメッセージが視覚/非視覚双方で認識できるか
- モーダルやメニューでフォーカス遷移が破綻していないか
- 名前・役割・値（Name/Role/Value）が推定可能か

### SEO詳細観点

- ページ意図に沿った1ページ1主題の見出し構成か
- 重複コンテンツ相当で canonical 等の意図が明確か
- 主要リンクがクロール可能な形で置かれているか

## レビュー除外

- インデント・改行・属性並び順の微差分
- 文字揃え/空白調整などの整形指摘

## 出力テンプレ

- 位置: `file:line`（分かる範囲）
- 重要度: 重大 / 中 / 軽微
- 確度: 確定 / 推定 / 判定保留
- 現状:
- 問題:
- 修正案:

## 最小修正ガイド

- まず操作不能/到達不能を解消（キーボード、ラベル、フォーカス）
- 次に構造（見出し、ランドマーク、リンク/ボタン役割）を整える
- 最後にSEO補強（title/description、画像最適化、内部リンク）を行う

## 判定保留の例

- 装飾画像か情報画像かの意図が仕様書なしでは不明
- SPAルーティング仕様が不明でtitle更新責務を断定できない

## 重大度ヒント

- 重大: キーボード操作不能、必須代替テキスト欠落、SEOメタの致命的不備
- 中: 構造不整合や属性不備でUX/保守性が悪化
- 軽微: 将来改善が望ましいが実害が小さい項目
