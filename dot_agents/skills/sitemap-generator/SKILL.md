# Sitemap Generator

既存Webサイトをクロールし、ページ一覧をスプレッドシート取り込み可能なCSVで出力する。

## ワークフロー

1. ユーザーから対象URLを取得
2. `scripts/crawl_site.py` を実行（robots.txt確認 → sitemap.xml取得 → クロール）
3. 制限到達時はユーザーに通知して継続確認
4. CSV出力後、**動的/固定の判定をレビューして修正する**（後述）
5. 最終CSVをユーザーに渡す

## 実行方法

```bash
python3 scripts/crawl_site.py <URL> [options]
```

### オプション

| オプション | デフォルト | 説明 |
|---|---|---|
| `--max-depth` | 5 | 最大クロール深度 |
| `--max-pages` | 500 | 最大ページ数 |
| `--output` | sitemap.csv | 出力CSVパス |
| `--output-tree` | (なし) | ツリー形式Markdown出力パス |
| `--dynamic-threshold` | 5 | 同一パターンN件以上で動的ページとして省略 |
| `--include-dynamic` | off | 動的ページも個別出力 |
| `--concurrency` | 5 | 並列クロール数 |
| `--resume` | (なし) | 中断データのJSONパスから再開 |
| `--save-state` | crawl_state.json | 中断時の状態保存先 |

### 出力CSVカラム

URL, 第1階層, 第2階層, 第3階層, 第4階層, 第5階層, 第6階層, タイトル, ステータス, h1, メタディスクリプション, canonical, リダイレクト先, noindex, ページ種別, sitemap.xml, 備考

階層カラムはパスのセグメントを分割して格納。スプレッドシートでフィルタ・グルーピングしやすい形式。パス順にソート済み。

### ページ種別の自動分類

トップ / 一覧 / 固定ページ / 詳細ページ / フォーム / ページネーション / 動的ページ

## 自動処理

### robots.txt

クロール前に `robots.txt` を取得し、`Disallow` パスを自動除外する。`Sitemap:` エントリがあれば sitemap.xml も自動取得する。

### sitemap.xml との突合

sitemap.xml のURL一覧とクロール結果を突合する:
- sitemap.xml にあるがクロールで未到達 → CSVに「sitemap.xmlに記載あるがクロールで未到達」と備考付きで追記
- sitemap.xml にあるが404 → 「sitemap.xmlに記載あるが404」

### SEO関連情報

- `<link rel="canonical">` → canonical カラムに記録
- リダイレクト（301/302）→ リダイレクト先カラムに最終URLを記録
- `<meta name="robots" content="noindex">` → noindex カラムにフラグ

### リンク切れ検出

ステータス 404/410/0（接続不可）のページをリンク切れとして検出し、参照元URLとともに備考に記録。サマリーでも一覧表示。

## 重要ルール

### 制限到達時の通知

クロール結果のstdoutにJSON警告が出た場合、**必ずユーザーに伝えて継続するか確認する**:

- 深度制限到達 → 「深度5まで取得しました。より深い階層も必要ですか？」
- ページ数上限到達 → 「500ページに達しました。上限を増やしますか？」

### 動的ページの省略

スクリプトはURLパターンからベストエフォートで動的ページを検出するが、誤判定がある。CSV出力後に以下の手順でレビューする。

#### レビュー手順

1. CSVを読み、「ページ種別」カラムを確認
2. 以下の判断基準で動的/固定を再判定し、CSVの「ページ種別」を修正

#### 動的/固定の判断基準

**動的ページ**（量産されるテンプレートページ。省略対象）:
- ブログ・ニュースの個別記事 → タイトルが具体的な記事見出し（例:「杉村病院が『よかボス宣言』を行いました！」）
- 商品詳細、イベント詳細、求人詳細
- ページネーション（`/page/2/`, `/page/3/`）
- 特徴: 同じ親パス配下に大量の兄弟がいて、タイトルがすべて異なる

**固定ページ**（サイト構造を成すページ。個別表示対象）:
- カテゴリ一覧ページ → タイトルが「お知らせ - メディア」「お知らせ - 重要」のような分類名
- 診療科・部門ページ → タイトルが「循環器内科」「リハビリテーション科」のような固有名
- 案内・説明ページ → タイトルが「初診の方」「入院のご案内」のような機能説明
- フォーム、サイトマップ、プライバシーポリシー
- 特徴: 親パス配下の兄弟数が少なく、タイトルがそれぞれ異なる役割を持つ

#### 修正方法

誤判定を見つけたら、CSVファイルを直接編集する:
- 固定ページなのに省略されていた → `--include-dynamic` で再実行してCSVから該当行を固定ページとして残す
- 動的ページなのに個別表示されていた → 該当行を削除し、親パスのサマリー行の件数を加算

ユーザーが動的ページも全件欲しい場合は `--include-dynamic` を付ける。

### 中断と再開

- Ctrl+C で中断すると `--save-state` に状態をJSON保存
- `--resume <JSONパス>` で中断地点から再開可能
- 大規模サイトで途中停止しても作業が無駄にならない

### エンコーディング

CSV は UTF-8 BOM 付き（`utf-8-sig`）で出力。Excel/Google Sheets で文字化けしない。

### 末尾スラッシュ正規化リダイレクトの処理

多くのサイトは `/services` → `/services/`（またはその逆）の 301 リダイレクトを大量に返す。これらはサイト構造上の重複であり、以下のように処理する:

1. リダイレクト元URLとリダイレクト先URLの**末尾スラッシュを除去して比較**
2. 一致すれば「末尾スラッシュ正規化」と判定し、**リダイレクト先URLを正規URLとして採用**
3. リダイレクト元の行はCSVから除外（重複を避ける）
4. ただし、リダイレクト元にタイトル・h1等のメタ情報がある場合はそちらを正規行に引き継ぐ

スクリプトの生出力では「一覧」に分類されたリダイレクト行が大量に混入するため、**必ずこの正規化処理を後処理で行うこと**。

### `/./` パスの除外

一部サイトでは `https://example.com/./blog/` のような相対パス残骸URLがクロールされる。これらは正規化後の `/blog/` と重複するため、**第1階層が `.` の行は除外する**。

### ページ数上限到達時の判断

上限到達の警告が出た場合、**まず未到達URLの内訳を分析する**:

```python
# 未到達URLの第1階層分布を集計
grep 'sitemap.xmlに記載あるがクロールで未到達' sitemap.csv | awk -F',' '{print $2}' | sort | uniq -c | sort -rn
```

大半が `/blog/` や `/news/` 等の動的ページであれば、追加クロールは不要と判断してよい。ユーザーには内訳を示して確認する。

### CSV後処理の完全手順

スクリプト実行後、以下の順序で後処理を行う:

1. **末尾スラッシュリダイレクトの除外** — 上記「正規化リダイレクトの処理」
2. **404行の除外** — リンク切れは別途報告済みなのでCSVから除外
3. **未到達行の除外** — 「sitemap.xmlに記載あるがクロールで未到達」の行を除外
4. **`/./` パスの除外** — 相対パス残骸の重複行を除外
5. **ページ種別の再判定** — 動的/固定の判断基準に基づきレビュー・修正
6. **動的ページの省略行マージ** — スクリプト生成の省略行と後処理追加分が重複する場合、件数を合算して1行にまとめる
7. **最終検証** — 省略行の重複なし、ページ種別空なし、200でタイトル空なしを確認

### 最終検証チェックリスト

CSV出力後、以下を必ず確認する:

- [ ] 動的ページ省略行に重複がないこと
- [ ] ページ種別が空の行がないこと
- [ ] ステータス200でタイトルが空の行がないこと
- [ ] 末尾スラッシュリダイレクトが除外されていること
- [ ] `/./` パスの重複行が除外されていること
